## 1. Nearest Neighbor 분류기 (최근접 이웃 분류기)

### 1.1 개요
**Nearest Neighbor (NN) 분류기**는 이미지 분류의 가장 기본적인 접근 방식입니다. 인간의 뇌는 이미지를 보는 순간 직관적으로 사물을 인식하지만, 컴퓨터는 이미지를 거대한 3차원 숫자 배열(예: CIFAR-10 이미지의 경우 $32 \times 32 \times 3$)로 인식합니다.

NN 분류기는 별도의 복잡한 학습 없이 **모든 학습 데이터(이미지와 라벨)를 그대로 메모리에 저장**합니다. 이후 새로운 테스트 이미지가 들어오면, 저장된 모든 학습 이미지와 하나씩 비교하여 **가장 유사한(거리가 가장 가까운) 단 하나의 이미지 라벨**을 정답으로 출력합니다.

### 1.2 시간 복잡도 (Computational Complexity)
Nearest Neighbor 분류기의 시간 복잡도는 다음과 같이 정의됩니다:

| 단계 (Phase) | 시간 복잡도 | 설명 |
| :--- | :--- | :--- |
| **학습 (Training)** | $O(1)$ | 학습 데이터를 메모리에 단순히 저장(암기)하면 끝남. |
| **예측 (Prediction)** | $O(N)$ | 테스트 이미지 1개를 예측하기 위해 모든 학습 데이터 $N$개와 대조해야 함. |

#### 💡 CS/AI 관점에서의 치명적인 단점
컴퓨터 과학 및 실무 딥러닝에서는 **학습 시간이 오래 걸리더라도, 예측(추론) 시간은 눈 깜짝할 사이에 끝나야 하는 알고리즘**을 선호합니다. 모델을 스마트폰이나 저전력 임베디드 기기에 탑재했을 때 즉각 연산이 가능해야 하기 때문입니다. 하지만 NN 분류기는 정확히 그 반대로 작동하므로 **실제 서비스나 실무에서는 사용할 수 없습니다.**

## 2. 거리 측정 공식: L1 Distance vs. L2 Distance

두 이미지 사이의 "유사도"를 측정하려면 수학적 기준(거리)이 필요합니다. CS231n 강의에서는 대표적인 두 가지 기하학적 거리 공식을 소개합니다.

### 2.1 L1 Distance (맨해튼 거리 - Manhattan Distance)
L1 거리는 두 이미지의 동일한 위치에 있는 픽셀 값들의 **절대 차이의 총합**을 구합니다.

$$d_1(I_1, I_2) = \sum_{p} |I_1^p - I_2^p|$$

* **기하학적 특징:** 좌표축의 회전에 영향을 받습니다. 즉, 좌표계를 회전하면 점들 사이의 L1 거리 값이 변합니다.
* **사용 시점:** 입력 특징(Feature)들이 각각 독립적이고 고유한 의미를 가질 때(예: 방의 개수, 직원의 수 등 서로 섞이지 않는 독립된 카테고리 속성일 때) 주로 사용합니다.

### 2.2 L2 Distance (유클리드 거리 - Euclidean Distance)
L2 거리는 두 이미지의 픽셀 값 차이를 **제곱하여 모두 더한 뒤, 제곱근(루트)**을 씌워 구합니다. 흔히 말하는 기하학적 직선거리입니다.

$$d_2(I_1, I_2) = \sqrt{\sum_{p} (I_1^p - I_2^p)^2}$$

* **기하학적 특징:** 회전 불변성(Rotation-invariant)을 가집니다. 좌표계를 아무리 회전해도 두 점 사이의 실제 직선거리는 변하지 않습니다.
* **사용 시점:** 특징들이 연속적인 기하학적 공간에 속해 있거나, 각 요소가 구체적으로 어떤 의미인지 모르는 일반적인 상태에서 요소 간의 범용적인 물리적 거리를 구하고자 할 때 선호됩니다.

## 3. 하이퍼파라미터와 k-Nearest Neighbor (kNN)

### 3.1 하이퍼파라미터(Hyperparameter)란?
가중치(Weights)처럼 모델이 데이터로부터 스스로 학습하는 파라미터와 달리, **하이퍼파라미터는 개발자가 학습 시작 전에 직접 지정해 주어야 하는 설정 값**입니다. (예: kNN에서 몇 개의 이웃을 볼 것인가($k$), 거리는 L1을 쓸 것인가 L2를 쓸 것인가 등)

### 3.2 k-Nearest Neighbor (kNN)
단 하나의 가장 가까운 이웃만 보는 대신, 상위 **$k$개의 가장 가까운 이웃**을 찾아 그 이웃들의 라벨 중 **다수결 투표(Majority Vote)**로 최종 라벨을 결정하는 방식입니다.
* $k$ 값을 조절하면 데이터 내부의 소음(Noise)이나 이상치(Outlier)에 강인해지며, 결정 경계(Decision Boundary)가 부드러워지는 효과를 얻을 수 있습니다.

### 3.3 하이퍼파라미터 튜닝 방법 (데이터 분할)
하이퍼파라미터를 조정할 때 **절대로 테스트 세트(Test set)를 사용하면 안 됩니다.** 테스트 세트에 맞춰 하이퍼파라미터를 고르면 모델이 테스트 세트에만 과적합(Overfitting)되어, 실제 새로운 데이터가 들어왔을 때 성능이 엉망이 됩니다.

1. **Train / Validation / Test 분할:** 데이터를 3가지로 나눕니다. *학습 세트(Train)*로 모델을 공부시키고, *검증 세트(Validation)*로 어떤 하이퍼파라미터($k$값 등)가 가장 좋은지 시험합니다. 마지막으로 확정된 최고의 모델을 딱 한 번 *테스트 세트(Test)*로 최종 평가합니다.
2. **교차 검증 (Cross-Validation):** 데이터셋의 크기가 너무 작을 때 주로 씁니다. 학습 데이터를 $V$개의 구간(Fold)으로 쪼갠 뒤, 번갈아가며 1개 구간을 검증 세트로 쓰고 나머지를 학습 세트로 사용하여 평균 성능을 냅니다. (다만, 딥러닝은 데이터가 워낙 방대하고 연산 비용이 크기 때문에 교차 검증은 거의 쓰지 않고 고정된 Validation 세트를 씁니다.)

## 4. 선형 분류기 (Linear Classifier): 현대 딥러닝의 초석

선형 분류기는 향후 배울 인공신경망(Neural Networks)과 합성곱 신경망(CNN)을 구성하는 가장 기본적이고 강력한 빌딩 블록입니다.

### 4.1 스코어 함수와 플래툰 (Flattening)
이미지를 선형 모델에 입력하기 위해서는 먼저 3차원 이미지 행렬을 하나의 긴 1차원 열 벡터 $x$로 **한 줄로 펼치는 작업(Flatten)**을 거쳐야 합니다. (행렬 크기: $[D \times 1]$, 여기서 $D = 가로 \times 세로 \times 채널$)

선형 스코어 함수는 이미지의 모든 픽셀에 가중치를 곱하고 바이어스를 더해 각 클래스별 점수를 계산합니다:

$$f(x, W, b) = Wx + b$$

* $x$: 1차원으로 길게 펼쳐진 이미지 입력 벡터 (크기: $[D \times 1]$)
* $W$: 가중치 행렬 (Weight Matrix, 크기: $[C \times D]$, $C$는 분류할 클래스의 총 개수). 가중치 행렬의 각 행은 특정 클래스에 대한 '템플릿(Template)' 역할을 합니다.
* $b$: 바이어스 벡터 (Bias Vector, 크기: $[C \times 1]$). 입력 이미지와 상관없이 특정 클래스에 가지는 정적인 선호도(사전 확률)를 나타냅니다.

## 5. 손실 함수 (Loss Function)와 소프트맥스 (Softmax)

현재 설정된 가중치 $W$와 바이어스 $b$가 얼마나 예측을 못 하고 있는지 정량적으로 정답과 비교하여 틀린 정도를 계산하는 척도가 바로 **손실 함수(Loss Function)**입니다.

### 5.1 소프트맥스 분류기 (Softmax Classifier)
소프트맥스 분류기는 이진 분류 알고리즘인 로지스틱 회귀를 다중 클래스 분류로 확장한 것입니다. 모델이 출력한 가공되지 않은 스코어 값($s = f(x_i, W)$)에 지수함수($e$)를 취해, **총합이 1이 되는 형태의 확률 분포**로 변환해 줍니다.

특정 클래스 $k$에 대한 소프트맥스 확률 공식은 다음과 같습니다:

$$P(Y = k \mid X = x_i) = \frac{e^{s_k}}{\sum_{j} e^{s_j}}$$

### 5.2 크로스 엔트로피 손실 (Cross-Entropy Loss / 음수 로그 손실)
모델이 예측한 확률이 실제 정답 라벨과 얼마나 일치하는지 평가하기 위해 **음수 로그(Negative Log)**를 취합니다. 모델이 정답 클래스의 확률을 1(100%)에 가깝게 맞출수록 손실(Loss)은 0에 수렴하고, 정답 확률을 낮게 예측할수록 손실 값은 무한대로 발산합니다.

하나의 이미지 데이터 $i$에 대한 손실 함수 $L_i$의 식은 다음과 같습니다:

$$L_i = -\log \left( \frac{e^{s_{y_i}}}{\sum_{j} e^{s_j}} \right)$$

여기서 $s_{y_i}$는 데이터 $i$의 실제 정답 클래스($y_i$)가 받은 스코어 점수입니다.

### 5.3 전체 데이터셋의 최종 손실 (Total Loss)
전체 데이터셋에 대한 최종 손실은 모든 개별 데이터 손실의 평균을 구한 값에, 모델이 특정 데이터에 과도하게 치우치는 것을 방지하는 **규제화 항(Regularization Penalty)**을 더해서 계산합니다:

$$L = \frac{1}{N} \sum_{i=1}^{N} L_i + \lambda R(W)$$
